## Decisions Made

| Decision | Rationale |
|----------|-----------|
| CSS custom properties in App.css | Single source of truth for the color palette; all component CSS inherits via `var()` |
| Separate `.css` files per component | Keeps styles colocated with components; no CSS-in-JS dependency needed |
| `.card-section` wrapper divs in App.tsx | Allows a clean border divider between form and results without adding wrapper elements inside the components themselves |
| Results restructured into 4 sections | Clearer information hierarchy (Details / Cost Breakdown / Suggested Pricing / Margins) vs. one flat list |
| Section heading "Details" (not "Product") | Avoid duplicate `/product/i` match — both `<h3>Product</h3>` and `<dt>Product Name</dt>` would have broken the existing test |
| Button text "Calculate Pricing" | More descriptive; still matches the `/calculate/i` regex used in tests |
