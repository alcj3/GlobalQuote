## Decisions made

| Decision | Rationale |
|----------|-----------|
| Drop Groq + USITC entirely, use category map only | USITC endpoint was CORS-blocked; Groq added latency and an API key dependency; map is instant, reliable, and sufficient for demo |
| `Promise.race` for hard timeout (pipeline continues in background) | No need to thread `AbortSignal` through both Ollama calls; for a demo tool, UI unblocking is what matters |
| `warmOllama` is best-effort (silently swallows errors) | Warm-up failure should never surface to the user; if Ollama is down, the user will see the error on submit instead |
| Confidence section removed from UI entirely | Score (0â€“100) can't be justified to a non-technical audience; label without score adds noise without signal |
| Clothing floor $19/$20 â€” same 1-dollar gap pattern as home_goods | Consistency; the gap is intentional (gives the model a small buffer above the floor) |
| Two explicit margin direction statements (retail + supplier) | Original wording was vague enough for the model to hallucinate direction; explicit three-way conditionals with interpolated numbers remove ambiguity |
